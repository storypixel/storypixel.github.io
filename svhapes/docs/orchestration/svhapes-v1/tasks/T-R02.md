# T-R02 — Catalog taxonomy and agent contract critique

## Outcome

Recommend a coherent 18-shape v0.1 taxonomy and a minimal machine-readable interface that helps coding agents select and integrate shapes reliably.

## Inputs

- Svhapes `BRIEF.md`, `PRD.md`, and `DECISIONS.md`
- The Klerb organic/golden-ratio contour concepts from the current conversation

## Ownership

- **Owned files:** Read-only
- **Do not modify:** All repositories and external services
- **Integration owner:** Main agent

## Constraints and non-goals

- Do not implement or generate CSS.
- Avoid vague AI marketing; propose deterministic fields, commands, and selection guidance.

## Contract

### Consumes

- Accepted v0.1 scope and planned catalog size.

### Provides

- Shape families/names/tags and a recommended JSON/CLI/prompt contract.

## Acceptance criteria

- [ ] At least 18 distinct, understandable catalog concepts.
- [ ] Stable metadata fields and example agent selection logic.
- [ ] No unrelated changes.

## Validation

- **Focused:** Check taxonomy for duplication and coverage.
- **Integration:** Recommendations map directly to definitions and generated catalog fields.
- **Visual evidence:** Not required.

## Handoff

Report the proposed catalog, agent contract, tradeoffs, and unresolved risks. Do not mutate local or external state.
